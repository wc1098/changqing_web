# 东方视野影业官网 - 生产部署与云资源配置指南

本文档列举了项目在上线前，需要在阿里云控制台以及生产服务器上准备和配置的全部云资源与环境变量。

---

## 1. 阿里云 OSS 对象存储配置

短剧的海报、视频切片流等静态资源不能存在应用服务器本地，必须托付给对象存储（OSS），以保证并发性能和高吞吐率。

### 1.1 创建 OSS Bucket
* **地域选择**：建议选择 **阿联酋（迪拜）** 节点（`me-east-1`），以确保中东和北非当地客户的极速响应，并符合迪拜总部定位。
* **文件存储目录划分**：
  * `/media/` (公共读 Bucket)：存放海报图片、Logo、宣传照等。读写权限（ACL）设为 **公共读（Public Read）**。
  * `/videos/` (私有 Bucket)：存放短剧正片 MP4、HLS 分片（.m3u8 和 .ts）。读写权限（ACL）必须设为 **私有（Private）**，防止外界直连盗取。

### 1.2 获取 RAM 授权子账号
不要使用主账号的 AccessKey。请在阿里云 RAM 控制台创建一个子账号：
* **权限策略**：分配 `AliyunOSSFullAccess` 权限。
* **保存凭证**：获取该子账号的 `AccessKey ID` 和 `AccessKey Secret`，用于后续服务器挂载或 Payload CMS 插件上传。

## 2. 阿里云媒体处理 (MPS) 与视频转码配置

为了适配中东各地区参差不齐的移动网络环境，用户上传的原始视频（通常为 MP4 格式）必须转码为自适应码率的 **HLS 格式（.m3u8 + .ts 切片）**。

* **转码触发器配置**：
  * 在阿里云控制台开通 **媒体处理服务 (MPS)**。
  * 配置工作流：当有视频上传至 OSS 输入 Bucket 的指定目录时，自动触发转码。
  * **输出配置**：转码输出为 HLS 格式，保存至输出 Bucket，转码成功后触发 Webhook 告知 CMS 更新 `VideoAssets` 的转码状态为 `completed`。

## 3. 阿里云 CDN 内容分发网络配置

由于视频播放带宽需求高，直接穿透到 OSS 会产生昂贵的下行流量费且速度受限，必须配置 CDN 加速。

### 3.1 CDN 域名绑定
* **图片加速域名**：绑定如 `media.yourdomain.com`，回源地址指向公共读的 OSS 域名。
* **视频加速域名**：绑定如 `play.yourdomain.com`，回源地址指向私有的 OSS 域名。

### 3.2 视频 CDN 安全防盗链配置 (URL 鉴权)
对于私有视频流，必须在 CDN 域名控制台开启 **“URL 鉴权”**（建议使用 **A类鉴权**）：
* **原理解析**：当客户端请求播放视频时，后端 CMS 会通过鉴权密钥（PrivateKey）和时间戳生成临时的 CDN 签名链接。
* **配置参数**：
  * **主/备鉴权密钥**：在 CDN 侧随机生成一个 32 位的密钥，必须与生产环境中服务器 `.env` 的配置一致。
  * **有效时长**：建议设置为 1800 秒（30分钟）或 3600 秒（1小时）。过期后链接自动失效，防御盗链效果最佳。

## 4. 生产服务器环境变量配置 (`.env`)

在应用根目录下创建生产 `.env` 文件，需要配置以下变量：

```env
# 1. 基础应用配置
PORT=3000
NODE_ENV=production
PAYLOAD_SECRET=随机生成的64位强密钥(如openssl rand -hex 32)
NEXT_PUBLIC_SERVER_URL=https://www.yourdomain.com

# 2. 数据库配置（生产环境 PostgreSQL）
DATABASE_URI=postgresql://db_user:db_password@db_host:5432/db_name

# 3. 阿里云 OSS 配置（用于海报/文档上传）
ALIYUN_OSS_ACCESS_KEY_ID=您的RAM子账号AccessKey
ALIYUN_OSS_ACCESS_KEY_SECRET=您的RAM子账号Secret
ALIYUN_OSS_BUCKET=eastern-vision-media-public
ALIYUN_OSS_ENDPOINT=oss-me-east-1.aliyuncs.com
ALIYUN_OSS_REGION=me-east-1

# 4. 视频签名与鉴权配置
# 用于对接 CDN URL 鉴权 A 方式的密钥，生成防盗链临时 URL
CDN_PRIVATE_KEY=您在阿里云CDN控制台配置的URL鉴权密钥
CDN_PLAY_DOMAIN=https://play.yourdomain.com
```

## 5. 服务器反向代理 (Nginx) 与 SSL 配置

生产环境的 Node.js/Next.js 服务一般运行在本地 `3000` 端口，需配置 Nginx 进行转发。

* **SSL 证书申请**：在阿里云控制台申请免费的 SSL 证书（有效期 3 个月，支持自动续期），或者使用 Let's Encrypt。
* **Nginx 配置要点**：
  * 强制将所有 HTTP (`80` 端口) 流量重定向至 HTTPS (`443` 端口)。
  * 配置 `client_max_body_size 500M`，以防管理员上传大型视频资产或高清海报时被 Nginx 默认的 1M 限制拦截。
  * 开启 Gzip 压缩，优化 Next.js 静态文件传输速度。
